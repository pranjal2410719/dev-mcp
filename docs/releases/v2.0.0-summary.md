# 📢 dev-mcp v2.0.0 — AI Engineering OS Release Summary

`dev-mcp` v2.0.0 is now officially released! This major upgrade transitions `dev-mcp` from a context-management server into a stateful, secure **AI Engineering Operating System** built for multi-workspace isolation, path jailing, and Open Design integrations.

---

## 🔍 What Changed & Why It Matters

- **Workspace Lock**: Enforces absolute path boundaries, jailing all file/shell executions to the active project root directory when `safe`/`trusted` safety modes are active. This prevents AI agents from modifying system-level files or escaping workspaces.
- **Safety Modes (`safe`, `trusted`, `lab`)**: Configurable workspace safety limits that determine what shell commands or destructive file actions are permitted.
- **Automated Workspace Orchestration**: Spawn isolated project workspaces using `create_project_workspace` with predefined PRD/context templates (`nextjs`, `python`, `blank`), complete with independent git repositories.
- **Open Design Support**: Standardized context metadata blocks to trace and audit design artifacts and prototype assets in generated projects.

## ⚙️ Upgrade Notes

To upgrade to `v2.0.0` from a previous installation:
1. Pull the latest commits from `main`.
2. Re-install editable dependencies:
   ```bash
   pip install -e .
   ```
3. Initialize or adopt workspaces to bootstrap the new `.project_brain/config.yaml` config (defaulting to `safety_mode: safe`).
